# Publish Dev6 Release

Publish from the `getjusto/dev6` repository only. Use the existing repo release flow instead of inventing a new one.

The canonical release command is `pnpm release:mac`. It uses `scripts/publish-mac.sh`, signs with the local keychain identity, notarizes with Apple credentials loaded from `scripts/release-env.local.sh`, publishes artifacts to GitHub Releases, and usually leaves the release as a draft that must be published with `gh`.

## Preconditions

- Work in `/Users/nicolaslopezj/Code/Projects/justo/dev6`.
- Keep `scripts/release-env.local.sh` local and ignored by git. Never print or commit its secret values.
- Require these env vars at runtime:
  - `APPLE_ID`
  - `APPLE_APP_SPECIFIC_PASSWORD` or `APPLE_PASSWORD`
  - `APPLE_TEAM_ID`
- Allow `GH_TOKEN` to come from `gh auth token` unless already exported.
- Use signing identity `Orionsoft SpA (3CZ24HA8DS)` unless the user explicitly changes it.

## Release Workflow

1. Confirm the repo is clean with `git status --short`.
2. Inspect the current version with `node -p "require('./package.json').version"`.
3. If the user wants a new version, update `package.json` first, commit the bump, and push `main` before publishing.
4. Run `pnpm build` if you need a fast preflight. Prefer going straight to `pnpm release:mac` when the repo is already known-good.
5. Run `pnpm release:mac`.
6. Wait through signing and notarization. Apple notarization can take minutes with no output.
7. Verify the release with `gh release view v<version> --repo getjusto/dev6 --json url,isDraft,tagName,assets`.
8. If `isDraft` is `true`, publish it with:
   - `gh release edit v<version> --repo getjusto/dev6 --draft=false`
9. Confirm the final public release URL and that expected assets exist:
   - `.dmg`
   - `-mac.zip`
   - `latest-mac.yml`
   - blockmap files
10. Confirm the working tree is still clean.

## Required Repo Knowledge

- Release config lives in `electron-builder.yml` and publishes to GitHub Releases.
- The release script is `scripts/publish-mac.sh`.
- Local Apple secrets live in `scripts/release-env.local.sh` and must stay out of git.
- The repo is public and releases are public.
- Auto-update assets are generated by `electron-builder`; do not handcraft `latest-mac.yml`.

## Safety Rules

- Never paste or echo secret values in user-facing output.
- Never commit `scripts/release-env.local.sh`.
- Never rewrite an existing published release to sneak in code changes. Ship a new version instead.
- If the user asks to publish after code changes, make sure those changes are committed and pushed first so the release matches `main`.
- If a release fails mid-flight, inspect the failure before retrying. Common checkpoints are certificate naming, notarization wait time, and draft release state.

## Common Commands

```bash
git status --short
node -p "require('./package.json').version"
pnpm release:mac
gh release view v0.1.1 --repo getjusto/dev6 --json url,isDraft,tagName,assets
gh release edit v0.1.1 --repo getjusto/dev6 --draft=false
```

Replace `v0.1.1` with the actual version being shipped.

## Response Pattern

- State the target version and whether it was bumped in this run.
- State whether the release was notarized successfully.
- Include the GitHub release URL.
- Mention any residual warnings, such as missing `author` in `package.json`, only if they appeared during the release.
